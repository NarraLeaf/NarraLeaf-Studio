## WI-1 接口 diff 盘点结论

- **版本真相**：卡面引 `dev_nomen 576015c`，但该提交是**中间态 0.16.0**（native SoftWipe/Blinds/SoftIris/MaskTransition，≈0.15 接口）＝此前 node_modules 里手拷的旧 dist。真正的「接口大改 + Mask 词表」在 dev_nomen HEAD（第二个 `publish:0.16.0` = `9450525`），且**npm `narraleaf-react@0.16.0` 即该 Mask 版**。故 WI-2 = `yarn add narraleaf-react@^0.16.0`，无需重建引擎、无需手拷 dist。576015c 是过期指针。
- **接口大改**（引擎 CHANGELOG [0.16.0]）：转场改为「引擎实例化 + 几何词表」。base `Transition`/`ImageTransition` 保留（自定义类仍可编译）。移除 `MaskTransition`、native `SoftWipe`/`SoftIris`/`Blinds`；新增/统一 `Reveal`(A→B 直切) + `ThroughColor`(过色) + `Mask` 静态词表 + native `Push`/`BlurDissolve`。`Dissolve`/`FadeIn` 位置参数→options 对象（`startPos`→`offset`）。
- **新转场枚举**（`Mask.*`，任意角度/羽化）：`wipe`、`barnDoor`、`iris`(circle/ellipse)、`clock`、`fan`、`dots`(stagger)、`blinds`；+`Mask.invert/toStyle`。
- **Studio 消费面**：`createTransition`（storyCompiler）；`transitions.ts` 词表（单一真源，specs 自动派生）；`StoryTransitionRef.kind` 类型；`ActionInspector` 的 `TransitionEditor`；i18n storyInspector。Story Motion 走 Transform 预设（另一子系统），不受影响。

## 两个 engine-defect 停机/裁决项

1. **slide / native Push（保留 Studio `Slide`）**：native `Push` 用 `vw/vh` 平移，但转场层堆栈 wrapper 是 `inset:0`（信箱化舞台盒尺寸，见 NLR `Image.tsx` stackStyle）。窗口宽高比 ≠ 设计比时（含 Studio Dev Mode 预览几乎任意面板尺寸）会溢出、露背景——正是 Studio 自带 `Slide` 用 `%` 规避的 bug。故 `slide` **保留 Studio `Slide`**（唯一保留的自定义转场，非 shim 而是绕障），其余自定义类全删。**建议**：引擎 `Push` 改 `%`（一行，照 Studio Slide）并发 0.16.1；届时删 Slide、slide→native Push。因无法发引擎版且 npm 0.16.0 会覆盖本地 dist 修改，Studio 侧绕障是当下唯一能保 slide 正确的路。
2. **darkness**：引擎 `Darkness` 存在但 0.16.0 **未导出**，无公开等价物（迁移前它也一直是未处理 no-op+警告，无回归）。保留规范值但编译期给显式诊断，交产品裁决（引擎导出 `Darkness`，或重派该词）。bible §1.3 已标 ⚠️。

## 包版本策略

`package.json ^0.15.0→^0.16.0`，`yarn.lock`/node_modules 均解析 npm `0.16.0`（＝Mask 版，含 mask/reveal/push d.ts）。现状：npm 0.16.0 即目标，**dev 与生产/打包同用 npm 版**，此前「dist 拷贝跟随 dev_nomen」的手拷 override 已不再需要（npm 已追上）。**唯一未合流点**＝上文 Push 缺陷：若采纳修 Push，需引擎发 0.16.1（否则 `yarn install` 会把带 `%` 修复的本地 dist 覆盖回 `vw/vh`）——**未擅改发布管线**，仅提方案。
